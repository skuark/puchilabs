# Tips para Puchita

Lista de consejos que Claude comparte de forma aleatoria al inicio de cada sesión.
Para añadir uno nuevo, sigue el formato: `- **[número]** Texto del tip.`

---

- **1** Un buen mensaje de commit describe *qué cambiaste y por qué*, no solo "cambios". Por ejemplo: `"Añado preguntas sobre sueños y futuro"` es mucho mejor que `"update"`.
- **2** Antes de escribir código, explícate en voz alta (o por escrito) qué quieres que haga. Muchas veces el propio razonamiento lleva a la solución sin necesidad de ayuda.
- **3** Los errores de consola no son el enemigo — son pistas. Lee el mensaje despacio antes de buscar ayuda. Casi siempre te dice exactamente dónde está el problema.
- **4** Guarda y recarga el navegador después de cada cambio pequeño. Cuanto antes veas el efecto, antes sabes si vas por buen camino.
- **5** `Ctrl+Z` (o `Cmd+Z`) es tu red de seguridad en el editor. Y Git es la red de seguridad grande: si haces commit con frecuencia, nunca puedes perder mucho trabajo.
- **6** Cuando algo no funciona, prueba a reducir el problema: comenta parte del código y comprueba si el error sigue. Así localizas exactamente qué línea falla.
- **7** Las DevTools del navegador (F12) son tu mejor herramienta. La pestaña *Console* muestra errores; la pestaña *Elements* te deja inspeccionar y editar el HTML y CSS en tiempo real.
- **8** Si llevas más de diez minutos bloqueada sin avanzar nada, pide ayuda. Atascarse mucho tiempo sin salir no es aprender — es frustrarse. Pedir ayuda a tiempo es una habilidad profesional.
- **9** Nombrar bien las variables importa más de lo que parece. `precioTotal` es infinitamente más claro que `pt` o `x`. El código lo leerás tú misma dentro de un mes.
- **10** No hace falta entender todo el código de un proyecto para poder trabajar en él. Localiza la parte que te afecta y céntrate en esa.
